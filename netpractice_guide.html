TCP/IP Subnetting 10 Levels
Conceitos Fundamentais
🌐

Endereço IP (IPv4)

32 bits divididos em 4 octetos (ex: 192.168.1.10). Identifica unicamente um dispositivo em uma rede. Intervalo: 0–255 por octeto.

🎭

Máscara de Sub-rede

Define qual parte do IP é a rede e qual é o host. Exemplo: 255.255.255.0 = /24 → os primeiros 24 bits são a rede.

🚪

Gateway Padrão

IP do roteador usado para sair da rede local. Todo dispositivo que precisa alcançar outra rede precisa de um gateway configurado.

🔀

Roteador

Conecta redes diferentes. Cada interface de um roteador pertence a uma rede diferente e tem seu próprio IP e máscara.

🔌

Switch

Conecta dispositivos dentro da mesma rede. Não separa redes — todos os dispositivos compartilham a mesma sub-rede.

🗺️

Tabela de Rotas

Mapeia destinos para o próximo hop. 0.0.0.0/0 é a rota padrão (default route) — envia tudo que não tem rota específica.

Regras de Ouro

🔵 Dispositivos no mesmo switch devem estar na mesma sub-rede.

🟣 Interfaces de roteador conectando redes diferentes devem estar em sub-redes diferentes.

🟢 O gateway de um dispositivo deve ser um IP da mesma sub-rede que o dispositivo.

🟠 Todo dispositivo que precisa sair da sua rede deve ter um gateway.

🔵 0.0.0.0/0 como destino na tabela de rotas = rota padrão.

🟡 Nunca use o endereço de rede (primeiro) ou broadcast (último) em um host.

🔴 IPs na mesma rede: apenas o último octeto (ou mais) muda — os bits de rede são iguais.

Soluções por Nível
01

Dois Hosts — Comunicação Direta

Sem roteadores. Dois hosts precisam se comunicar na mesma rede.

ℹ️ Dois hosts numa mesma rede devem ter IPs com o mesmo prefixo de rede e máscaras iguais.

Topologia

Host A (Client A)
IP104.96.23.12
Mask255.255.255.0
─────
Host B (Client B)
IP104.96.23.253
Mask255.255.255.0
1
Observe o IP já dado (Host A)

O Host A já tem IP 104.96.23.12. A rede é 104.96.23.0/24.

2
Configure o Host B na mesma rede

Escolha qualquer IP em 104.96.23.1 – 104.96.23.254 que não seja o do Host A. Exemplo: 104.96.23.253.

3
Máscaras iguais

Ambos devem ter máscara 255.255.255.0 (ou /24).

02

Dois Hosts — Mesma Sub-rede

Dois pares de hosts conectados diretamente. Cada par é uma rede isolada.

💡 A máscara /30 (255.255.255.252) permite apenas 2 hosts — útil para links ponto-a-ponto.

Topologia

Host A
IP192.168.98.2
Mask255.255.255.252
─────
Host B
IP192.168.98.1
Mask255.255.255.252
Host C
IP192.168.15.1
Mask255.255.255.252
─────
Host D
IP192.168.15.2
Mask255.255.255.252
1
Par A–B: use /30

Com máscara /30, só há 2 IPs utilizáveis. Coloque A em .1 e B em .2 (ou vice-versa) dentro do mesmo bloco.

2
Par C–D: rede diferente

Use um prefixo de rede diferente do par A–B para evitar conflitos.

3
Máscaras sempre iguais nos dois lados

Dois hosts se comunicando diretamente devem ter a mesma máscara.

03

Switch — Múltiplos Hosts

Um switch conecta 3+ hosts. Todos devem estar na mesma sub-rede.

ℹ️ Um switch não separa redes. Todos os hosts conectados a ele precisam estar na mesma sub-rede com a mesma máscara.

Topologia

Host A
IP104.198.241.125
Mask255.255.255.128
──[SW]──
Host B
IP104.198.241.126
Mask255.255.255.128
│
Host C
IP104.198.241.1
Mask255.255.255.128
1
Identifique a rede pelo host fixo

Se um IP já está definido (ex: 104.198.241.125/25), a rede é 104.198.241.0/25. IPs disponíveis: .1 até .126.

2
Configure todos os outros hosts na mesma rede

Todos os hosts no switch devem usar IPs do range 104.198.241.1–104.198.241.126 e máscara 255.255.255.128.

04

Roteador — Duas Redes

Um roteador conecta dois grupos de hosts em redes separadas.

⚠️ Cada interface do roteador deve estar em uma sub-rede DIFERENTE. Nunca use o mesmo prefixo de rede nos dois lados.

Topologia

Host A
IP85.17.5.4
Mask255.255.255.0
GW85.17.5.1
────
Router R (eth0)
IP85.17.5.1
Mask255.255.255.0
│ (eth1)
Host B
IP190.3.2.253
Mask255.255.255.0
GW190.3.2.254
────
Router R (eth1)
IP190.3.2.254
Mask255.255.255.0
1
Configure a interface do roteador (lado esquerdo)

O IP do roteador deve estar na mesma sub-rede que o Host A. Ex: Host A = 85.17.5.4/24 → Router eth0 = 85.17.5.1/24.

2
Configure a interface do roteador (lado direito) em outra rede

Use uma rede totalmente diferente. Ex: 190.3.2.0/24. Router eth1 = 190.3.2.254/24.

3
Configure o Gateway nos hosts

O gateway de cada host = IP da interface do roteador no mesmo lado. Host A → gateway 85.17.5.1. Host B → gateway 190.3.2.254.

05

Roteador + Tabela de Rotas

Hosts precisam de rotas configuradas para alcançar redes remotas via roteador.

💡 Nos hosts, use 0.0.0.0/0 como destino na tabela de rotas — isso significa "qualquer destino", e o próximo hop é o gateway (IP do roteador).

Configuração da Tabela de Rotas

Host A — Routing Table
Dest0.0.0.0/0
Next hop54.117.30.126 (IP do roteador)
Host B — Routing Table
Dest0.0.0.0/0
Next hop54.117.30.62 (IP do roteador)
1
Configure IPs dos hosts e interfaces do roteador

Assim como no Level 4 — cada lado do roteador é uma rede diferente.

2
Adicione rota padrão em cada host

Destino: 0.0.0.0/0. Next hop: IP da interface do roteador no mesmo lado do host.

3
Verifique rota no roteador (se necessário)

Alguns níveis exigem que o roteador também tenha rotas para alcançar as redes dos hosts.

06

Internet — Rota Padrão

A rede se conecta à internet. É necessário configurar rotas para o tráfego externo.

⚠️ A internet tem um IP fixo no diagrama. Você configura a rota do seu roteador para a internet, e a internet precisa saber como chegar à sua rede.

Topologia

Host A
IP10.0.0.2
Mask/24
GW10.0.0.1
────
Router R
eth010.0.0.1
eth1163.172.250.1
Route→0.0.0.0/0 → eth1
────
🌐 Internet
IP163.172.250.12
Route→10.0.0.0/24 → R
1
Configure a rede interna normalmente

Host A ↔ Router eth0: mesma sub-rede.

2
Rota do roteador para a internet

Na tabela de rotas do roteador: destino 0.0.0.0/0, next hop = IP da internet.

3
Rota da internet para a rede interna

Na tabela de rotas da internet: destino = sua rede interna (ex: 10.0.0.0/24), next hop = IP externo do seu roteador.

07

Múltiplos Roteadores

Dois roteadores interconectados. Rotas entre redes precisam cruzar dois saltos (hops).

💡 O link entre os dois roteadores é uma rede própria (geralmente /30). Cada roteador precisa de rotas para as redes que não estão diretamente conectadas a ele.

Topologia

Host A
IP192.168.0.2/24
GW192.168.0.1
──
Router A
eth0192.168.0.1/24
eth110.0.0.1/30
──
Router B
eth010.0.0.2/30
eth1172.16.0.1/24
──
Host B
IP172.16.0.2/24
GW172.16.0.1
1
Link entre roteadores (rede /30)

Router A eth1 = 10.0.0.1/30, Router B eth0 = 10.0.0.2/30. Essa é uma rede ponto-a-ponto.

2
Rotas no Router A

Para alcançar a rede de Host B (172.16.0.0/24), próximo hop = 10.0.0.2 (Router B).

3
Rotas no Router B

Para alcançar a rede de Host A (192.168.0.0/24), próximo hop = 10.0.0.1 (Router A).

4
Gateways dos hosts

Host A → gateway 192.168.0.1. Host B → gateway 172.16.0.1.

08

Switch + Roteador + Internet

Switch agrupando hosts, roteador e conexão com internet simultâneos.

ℹ️ Neste nível há 3 redes distintas: rede do switch (hosts), link roteador-internet, e a "rede internet". Cada uma precisa ter sub-rede separada.

Topologia

Host C
IP10.0.0.2/26
GW10.0.0.1
Host D
IP10.0.0.3/26
GW10.0.0.1
└──[SW]──┘
│
Router R
eth0 (SW)10.0.0.1/26
eth1 (INT)163.172.1.1/30
│
🌐 Internet
IP163.172.1.2/30
1
Rede do switch (/26 = 62 hosts)

Router eth0 + Host C + Host D todos na mesma /26. Ex: 10.0.0.0/26.

2
Link roteador → internet (/30)

Rede separada para o link ponto-a-ponto. Ex: 163.172.1.0/30.

3
Rotas de retorno da internet

A internet precisa de rota para a rede dos hosts: destino 10.0.0.0/26, next hop = IP externo do roteador.

09

Topologia Complexa — 3 Redes

Múltiplas sub-redes, dois roteadores, switch e internet. Exige planejamento cuidadoso.

⚠️ Certifique-se de que cada sub-rede usa um range de IPs completamente diferente. Sobreposição de sub-redes causa falha na comunicação.
1
Mapeie todas as redes presentes

Identifique quantas redes existem: rede A (switch1), rede B (switch2), link roteador1↔roteador2, link para internet. Cada uma precisa de uma sub-rede diferente.

2
Use /30 para links ponto-a-ponto

Entre roteadores e entre roteador e internet, use /30 para economizar IPs.

3
Configure rotas em ambos os roteadores

Cada roteador precisa saber como chegar às redes que não estão diretamente conectadas a ele. Use rotas específicas ou rota padrão.

4
Rota na internet

A internet precisa de rotas para TODAS as sub-redes internas, ou use um supernet que cubra todas (ex: 10.0.0.0/8 se todas usam esse prefixo).

5
Verifique os logs

Use os logs no bottom da interface do NetPractice para identificar o que ainda está faltando (gateway, rota incorreta, IP inválido).

10

Nível Final — Topologia Completa

O nível mais complexo: múltiplos switches, roteadores, sub-redes e internet integrados.

💡 Dica final: resolva de dentro para fora. Configure primeiro as redes locais (hosts + switches), depois os links entre roteadores, e por último as rotas para internet.
1
Desenhe o mapa em papel

Anote cada dispositivo, suas interfaces, e qual rede cada interface pertence. Isso evita erros de sub-rede sobreposta.

2
Defina blocos de endereçamento

Exemplo: use 10.0.1.0/24 para rede A, 10.0.2.0/24 para rede B, 10.0.3.0/30 para link entre roteadores, etc.

3
Configure hosts e interfaces de roteadores

Preencha IPs, máscaras e gateways nos hosts. Configure as interfaces dos roteadores com IPs das respectivas redes.

4
Configure todas as tabelas de rotas

Cada roteador precisa de rotas para redes que não conhece diretamente. Use 0.0.0.0/0 como rota padrão quando possível.

5
Configure rotas na internet

A internet deve ter rotas de retorno para cada sub-rede interna, ou um supernet (ex: 10.0.0.0/8).

6
Exporte antes de sair!

Clique em [Get my config] assim que o nível estiver completo. O arquivo exportado vai para o seu repositório Git.

📌 Lembre-se: durante a defesa você terá que resolver 3 níveis aleatórios com tempo limitado. Pratique até conseguir resolver cada nível em menos de 5 minutos.
